## 1. 모델 통계적 진단

### 1.1 잔차 자기상관 잔존 — Ljung-Box p<0.001
- **현상**: 차수 (1,1,2)(1,1,1,7)에서 잔차 LB p<0.001 — 잡지 못한 시간 구조 잔존
- **개선 방향**: 차수 확장
  - 후보: (2,1,2)(2,1,2,7), (3,1,3)(1,1,1,7), (2,1,3)(1,1,2,7)
  - 시계열 CV(expanding window 5-fold)로 안정성 검증
- **예상 효과**: 점예측 변화 미미 가능, 통계적 정당성 ↑
- **비용**: 차수 확장 학습 시간 2-3배, CV 추가 시 5배
- **위험**: 과적합 — AIC↓이지만 2025 MAE↑ 가능

### 1.2 잔차 비대칭 — skew -1.8, kurt 13.9 (log 스케일)
- **현상**: 음력 명절·태풍 등 일과적 충격에서 큰 음의 점프
- **개선 방향**:
  - 명절 더미 보강 (1.4 참조 — 사용자 보류)
  - 또는 ARMA-GARCH로 조건부 이분산 모델링
  - 또는 robust 추정 (학생 t 분포 가정)
- **예상 효과**: PI 신뢰도 향상, 점예측 영향은 작음
- **비용**: GARCH 도입 시 statsmodels 외 라이브러리 (arch) 추가

### 1.3 학습 적합도(AIC) vs 일반화(MAE) 갈림
- **현상**: v1 phase1이 AIC 더 작지만 2025 MAE는 v2/v3c가 작음
- **개선 방향**: 시계열 CV 도입으로 차수 선택 기준 통일
  - expanding window 5-fold, 각 fold MAE 평균으로 차수 비교
- **예상 효과**: 차수 선택 신뢰도 ↑, 보고서 통계적 정당성 ↑
- **비용**: 학습 시간 5배 (9개 그리드 × 5 fold = 45 fit)

## 2. 외생변수 측면

### 2.1 `pop_total` p=0.677 — 유의하지 않음
- **현상**: 16년간 인구 변동 폭 ±1.5%로 좁아 표준화 후 신호 약함
- **개선 방향**:
  - (a) 그냥 제거 — MAE 변화 작을 듯, 깔끔
  - (b) 가구수·1인가구 비율 등 보조 변수로 교체
  - (c) 차분 변수(`pop_total.diff()`) 시도
- **예상 효과**: 작음. 단순화 차원
- **비용**: 5-10분

### 2.2 희소 더미 OR 통합 후에도 비유의
- **현상**: `heat_th_2day_any` (n=110, p=0.37), `cold_th_2day_any` (n=15, p=0.24)
- **원인**: cdd_18·hdd_18·cdd_18²·hdd_18²가 이미 폭염·한파 효과 대부분 흡수
- **개선 방향**:
  - 임계 초과 정도(`max(ta_max - 33, 0)`) 같은 연속 변수로 대체
  - 또는 그냥 제거
- **예상 효과**: 작음

### 2.3 `ws_avg` 직접 효과 0
- **현상**: 풍속은 체감온도 산출 입력으로만 작동, 단독 회귀 효과 미미
- **개선 방향**: 제거 — 외생변수 14→13개
- **예상 효과**: MAE 변화 없음, 단순화

## 3. 예측 성능

### 3.1 9월 추석 후퇴 (v3c +1,668 MW)
- **현상**: 추석 연휴 직전 9월말 휴무 효과 미흡 — `is_holiday` 단일 더미 한계
- **개선 방향** (사용자 보류 결정):
  - `is_seollal`, `is_chuseok` 명절 당일 더미
  - `holiday_streak_len` 연휴 길이
  - `is_bridge_day` 샌드위치 데이
  - `days_to_next_holiday` 명절 직전·직후 거리
- **예상 효과**: 9월·1월·10월 worst case 5건 직접 개선 — MAE 200-500 추가 감소 가능
- **비용**: 30분 (calendar 변수 이미 일부 있음)

### 3.2 극단 폭염 8월 MAE 4,885 (v3c)
- **현상**: 비선형 cdd²로도 8월 개선 거의 없음 — 이미 v2부터 cdd·ma7 효과로 포화
- **개선 방향**:
  - `ta_max_above_35` 같은 극단부 더미
  - 또는 cubic spline (knot 25/30/35°C)
- **예상 효과**: 미상 — 8월 특히 어려운 구간

### 3.3 one-shot vs rolling 격차 (3.63% vs 1.87%)
- **현상**: 365일 누적 시 오차 2배 — 외생변수 미래값 가정 불완전
- **개선 방향**:
  - **외생변수 자체 예측 모델** (기상 forecast, 인구 추세)
  - 또는 점진적 horizon 평가 (7/30/90일 ahead)
- **예상 효과**: 실무 모델로 확장 시 필수, 학부 프로젝트로는 옵션

## 4. 실험 설계

### 4.1 시계열 CV 미수행
- **현상**: 2010-2024 전체 1회 학습 + 2025 1회 평가만
- **개선 방향**: expanding window — 각 fold에서 2010~2020/2021/.../2024 → 다음 1년 평가
- **예상 효과**: 차수·외생변수 선택 신뢰도 ↑
- **비용**: 학습 시간 5배

### 4.2 외생변수 실측치 사용 가정
- **현상**: 2025 one-shot 예측 시 2025 외생변수를 그대로 입력
- **시사**: 보고된 MAPE 3.63%는 "외생변수 완벽 예측 가정 하의 상한 성능"
- **개선 방향**:
  - 외생변수도 동시 예측 (기상은 KMA forecast API, 인구는 추세 ARIMA)
  - 또는 외생변수 노이즈 injection sensitivity test
- **예상 효과**: 실무 평가는 0.5-1.5%p MAPE 악화 예상

### 4.3 구조변화 더미 미투입
- **현상**: COVID(2020-02 ~ 2022-04), 폭염 기준 개정(2020-05), 한파 기준 개정(2017) 미반영
- **개선 방향**: `event_covid`, `regime_heat_alert_2020`, `regime_cold_alert_2017` 추가
- **예상 효과**: 잔차 분산 ↓ 가능, 2025 평가는 영향 작음(평가 기간 외)

## 5. 모델 확장

### 5.1 다른 모델 비교 (베이스라인 강화)
- **후보**:
  - TBATS — 다중 계절성 (주간 + 연간)
  - Prophet — 명절·휴일 자동 처리
  - LightGBM/XGBoost — 비선형, 변수 상호작용 자동
  - 신경망 (LSTM, N-BEATS, TFT) — 만약 시간 여유
- **예상 효과**: 보고서 풍부, ARIMAX보다 좋을 경우 분석 방향 흔들림
- **비용**: 모델당 2-4시간

### 5.2 앙상블
- **방향**: ARIMAX + Prophet 평균 또는 stacked
- **예상 효과**: MAE 추가 5-10% 감소 가능
- **비용**: 5.1 후속 작업

## 보고서 한계점 섹션에 그대로 쓸 문장 (보류 항목용)

> 본 모델의 한계는 다음과 같다.
> (1) 잔차에 자기상관이 잔존(Ljung-Box p<0.001)하여 본 차수보다 풍부한 구조의
> 모델(GARCH·고차 SARIMA)이 추가 정보를 추출할 여지가 있다.
> (2) 음력 명절(설·추석) 효과를 `is_holiday` 단일 더미로 흡수하기에는
> 다일 연휴의 누적 휴무 패턴이 복잡하여, 명절 인근 일자에서 ±10,000 MW
> 수준의 큰 오차가 관찰되었다.
> (3) 외생변수 일부(`pop_total`, `ws_avg`)는 16년간 변동 폭이 좁거나
> 체감온도 산출에 간접적으로만 기여해 단독 회귀 계수의 통계적 유의성이
> 낮았다.
> (4) 본 평가는 2025년 외생변수 실측치를 입력으로 가정한 상한 성능이며,
> 실무 적용 시 외생변수 자체의 예측 오차가 추가된다.
