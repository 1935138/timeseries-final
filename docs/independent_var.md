# 독립변수 (Exogenous Features) — 후보·출처·가공 메타

## 0. 문서 목적

본 문서는 데이터 **수집 이전 단계**의 메타 문서로, ARIMAX/SARIMAX 외생변수
후보를 카테고리별로 정리하고 **출처·정의·가공 방안**을 사전에 합의하기 위한
체크리스트다. 실제 수집된 데이터 명세·EDA 결과는 별도 산출물(`data/`,
`docs/research_log/`)에 기록한다.

- 종속변수 정합 기간: **2010-01-01 ~ 2025-12-31 (5,844일)**
- 종속변수 메타: `docs/dependent_var.md`
- 상위 계획: `docs/project_goals.md`
- 의사결정·수집 이력: `docs/research_log/`

## 1. 공통 원칙

1. **시간 정렬 기준**: 모든 변수는 KST 기준 일(day) 단위. 종속변수의
   `(년, 월, 일)`과 1:1 조인 가능해야 함.
2. **빈도 정합**:
   - 일 빈도(daily): 그대로 사용.
   - 월/분기 빈도: 발표일 기준 forward-fill + **발표 지연 lag** 명시
     (예: 산업생산지수는 익월 말 발표 → 최소 1~2개월 lag).
3. **미래 정보 누설 방지**:
   - 정규화/표준화 통계량은 **학습 구간(2010-2024)에서만** 계산하여
     2025 홀드아웃에 적용.
   - 사후 개정되는 통계(경제지표 등)는 발표 당시 값(vintage) 우선.
4. **결측치 처리**:
   - 기상 일자료 결측: 인접일 선형보간(연속 1~2일에 한함), 장기 결측은
     대체 관측소 사용 후 비고에 기록.
   - 더미: 결측은 0으로 처리하되 별도 마스킹 컬럼은 두지 않음(필요 시 명시).
5. **파생 변수 명명 규칙**: `<source>_<var>_<agg>` (예: `asos_tavg`,
   `kma_alert_heat_advisory`).
6. **구조변화 시점**은 별도 더미 변수로 명시(섹션 6).

## 2. 카테고리 A — 기상 (ASOS 일자료)

### 2.1 출처
- 기상자료개방포털 ASOS 일자료: https://data.kma.go.kr/data/grnd/selectAsosRltmList.do
- KOSIS `DT_1YL20651E` 주민등록인구(시도/시/군/구), 행정안전부, 연별: https://kosis.kr

### 2.1.1 대표화 방식 — 17개 시·도 인구 가중 평균
전국 단위 전력 수요와 정합시키기 위해 **17개 시·도 ASOS 대표 관측소**의
일자료를 **주민등록인구로 가중 평균**한다.

- **가중치 갱신 주기**: 매년 KOSIS 연도값(해당 연도 말 기준)으로 그 해 가중치 고정(연 1회 갱신).
- **세종특별자치시 처리**: 2012-07 출범으로 시계열 일관성을 위해
  **전 기간 충청남도에 통합**. 인구는 충남 + 세종 합산, 기상은 충남
  대표 관측소(천안 232) 사용.
- **시·도 ↔ 대표 관측소 매핑** 및 결정 경위: `docs/research_log/2026-05-25_대표관측소_인구가중평균.md`

가중 일자료 산식 (변수 `x` ∈ {tavg, tmax, tmin, rhum, wspd, srad}):

```
x_nationwide(t) = Σ_i w_i(year(t)) · x_i(t)
w_i(y) = pop_i(y) / Σ_j pop_j(y),  Σ_i w_i(y) = 1
```

여기서 `i`는 시·도(세종 통합 기준 16개 단위), `pop_i(y)`는 KOSIS 연별
주민등록인구의 y년 값(y-12-31 기준).

## 3. 카테고리 B — 기상 특보 (체감온도 기반 임계 더미)

기상청 공식 발효기준(2020년 폭염 개정·2017년 한파 개정 이후)에 정합한
**체감온도 기반 2일 연속 임계 더미 4개**로 단순화. 산출 절차·검증은
`docs/research_log/2026-05-26_0910_체감온도_산출_임계더미.md`.

### 3.1 (확정) 체감온도 기반 임계 더미 — 기상청 공식 발효기준 정합

전 기간 **동일 규칙**으로 산출 가능해 시계열 일관성이 높고, 기상청 공식
폭염·한파 ①조건(체감온도 33/35°C · 기온 -12/-15°C, 2일 연속)에 정합.

| 변수명 | 정의 | 단위 | 비고 |
|---|---|---|---|
| `heat_feels_th_2day_adv` | `feels_like_max ≥ 33°C` 2일 연속 | 0/1 | 폭염주의보 ①조건 |
| `heat_feels_th_2day_wrn` | `feels_like_max ≥ 35°C` 2일 연속 | 0/1 | 폭염경보 ①조건 |
| `cold_th_2day_adv` | `ta_hourly_min ≤ -12°C` 2일 연속 (10~4월) | 0/1 | 한파주의보 ②조건 |
| `cold_th_2day_wrn` | `ta_hourly_min ≤ -15°C` 2일 연속 (10~4월) | 0/1 | 한파경보 ②조건 |

**한파는 PT가 아닌 `ta_hourly_min` 사용**: 기상청 한파 ②조건은
"아침 최저 **기온**" 기준 — PT가 아닌 실측 기온.

### 3.2 제외된 정의

| 항목 | 제외 사유 |
|---|---|
| 폭염·한파 단일일 더미 (`heat_th_advisory` 등) | 공식 발효기준 모두 "2일 이상 지속" — 1일 단순 더미는 공식 비정합. §2.3 `cdd_18`·`cdd_18_ma3`가 1일 효과 흡수 |
| 폭염·한파 ②/③조건 ("급격한 상승/광범위한 피해") | 정성적 — 수식 불가 |
| 한파 ①조건 (전일대비 급강하 + 평년값 -3°C 이상 낮음) | 30년 평년 별도 산출 필요. ②조건만으로 한파 부하 점프 대부분 포착 |
| 공식 특보 (B) — 기상청 통보문 | (A) 임계 더미와 정보 중첩 90%+ 예상, 시·군·구 집계 비용 ↑ → Phase 2 검토 |

### 5.1 기본 변수 (Phase 1 포함)

| 변수명 | 정의·산식 | 단위 | 출처 | 빈도 | 가공방안 | 비고 |
|---|---|---|---|---|---|---|
| `ip_total` | 전산업생산지수(원지수) | 2020=100 | KOSIS `DT_1JH20201` (C1=0) | 월 → 일 | 발표 lag **2개월** 강제 후 ffill | 경기 사이클·COVID 충격·데이터센터 호황 모두 포착 |
| `pop_total` | 전국 주민등록인구 합계 | 명 | KOSIS `DT_1YL20651E` (시·도 합산) | 연 → 일 | 해당 연도값 ffill | 2019년 정점(51.85M) → 감소 전환 |

**가공 원칙**:
- `ip_total`: 통계청 산업활동동향은 익월 말 ~ 익익월 초 발표 → **2개월 lag**
  (M월 값은 M+2월 1일부터 사용). 시계열 초반 2개월(2010-01~02)은 첫 가용
  값으로 backfill (학습 시작 시점이므로 누설 아님).
- 계절조정지수 대신 **원지수**를 선택한 이유: 본 모델에는 이미 `month`,
  `dow`, `is_holiday`, 기상변수 등 계절 패턴을 잡는 변수가 풍부해
  계절조정 시 정보 중복 + sa 시리즈의 사후 개정 위험.
