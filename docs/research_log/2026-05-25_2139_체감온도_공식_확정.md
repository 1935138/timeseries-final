# Research Log

## 2026-05-25 — 체감온도(Perceived Temperature, PT) 공식 확정

### 목적

폭염·한파 특보 임계 더미를 **기상청 공식 기준 정합**으로 산출하기 위해
체감온도를 명시 계산한다. 기상청 폭염 특보(2020 개정)가 "일 최고 **체감온도**"
기준이므로, 실제 기온(`ta_max`) 단독 임계 더미는 신판 공식 기준과 정합 부족.

### 핵심 결정

1. **공식 출처**: 기상청 공식 체감온도 — 여름은 JMA·Steadman 기반 한국 보정식,
   겨울은 환경캐나다·NWS(2001) 공동 개정식 (기상청 동일 채택).
2. **산출 단위**: **시간별 체감온도 → 일 max(여름)·min(겨울) 집계** (공식 정의 그대로).
3. **계절 분리 적용**: 5/1~9/30 여름 공식, 10/1~4/30 겨울 공식.
4. **입력 데이터**: ASOS 시간자료 (`kma_sfctm2.php`) — 신규 수집 필요.

### 채택 공식

#### 여름 체감온도 (Heat Index, 5/1 ~ 9/30)

**Step 1 — 습구온도 Tw (Stull 2011 근사)**

입력: 기온 T (°C), 상대습도 RH (%)

```
Tw = T · atan(0.151977 · (RH + 8.313659)^0.5)
   + atan(T + RH)
   - atan(RH - 1.67633)
   + 0.00391838 · RH^1.5 · atan(0.023101 · RH)
   - 4.686035
```

(atan은 라디안 기준)

**Step 2 — 체감온도 PT (기상청 2020 개정판)**

```
PT_summer = -0.2442
          + 0.55399 · Tw
          + 0.45535 · T
          - 0.0022  · Tw^2
          + 0.00278 · Tw · T
          + 3.0
```

- 단위: °C
- 풍속은 여름 공식에 미포함.
- 한국 기상청 2016년 시범, 2020년 5월 폭염 특보 정식 적용.

#### 겨울 체감온도 (Wind Chill, 10/1 ~ 4/30)

입력: 기온 T (°C), 풍속 V (m/s)

**적용 조건**: T ≤ 10°C **AND** V ≥ 1.3 m/s. 그 외 `PT_winter = T`.

```
V_kmh = V × 3.6
PT_winter = 13.12
          + 0.6215 · T
          - 11.37  · V_kmh^0.16
          + 0.3965 · T · V_kmh^0.16
```

- 단위: °C
- 습도는 겨울 공식에 미포함.
- 환경캐나다·NWS 2001년 공동 개정식. 기상청 동일 채택.

#### 통합 적용 규칙

```
if 월 ∈ {5,6,7,8,9}:    PT = PT_summer(T, RH)
elif 월 ∈ {10,11,12,1,2,3,4}: PT = PT_winter(T, V)
```

→ **단일 컬럼 `feels_like` (시간별)** 산출 후, 일 단위에서:
- `feels_like_max`: 그날 시간별 PT의 최댓값
- `feels_like_min`: 그날 시간별 PT의 최솟값

### 정확도·근사 옵션 검토

| 방식 | 입력 | 정확도 | 채택 |
|---|---|---|---|
| A. 일평균 근사 | `ta_max + hm_avg`, `ta_min + ws_avg` | ±2~3°C 오차 | 미채택 |
| **B. 시간별 산출 후 일 집계** | 시간별 `ta, hm, ws` | **공식 정의 그대로** | **채택** |

**B 선택 이유**:
- 기상청 공식 특보는 "**시간별** PT 산출 후 일 중 max" 기준
- A는 일평균 습도가 일최고기온 시점(보통 오후·습도 최저) 습도와 다름 → 폭염일
  체감 2-3°C 과대평가
- 후속 학술 비교·논문 작성 시 공식 정의 정합 필수
- 데이터 수집 비용(2-4시간)은 1회성, 산출물 재사용성 높음

### 입력 데이터 요구

| 항목 | 필요 시간 해상도 | 출처 |
|---|---|---|
| 기온 (TA) | 시간별 | ASOS `kma_sfctm2.php` |
| 상대습도 (HM) | 시간별 | ASOS `kma_sfctm2.php` |
| 풍속 (WS) | 시간별 | ASOS `kma_sfctm2.php` |

**수집 규모**:
- 16개 시·도 대표 지점 × 192개월 × 일 24h × 약 28-31일/월
- 총 약 **2.24M 행** (≈ 1.2 GB)
- API 호출: 월 단위 × 16지점 × 192개월 ≈ 3,072회 (rate limit 고려 시 2-4시간)

### 가공 파이프라인 (5단계)

1. `src/collect_asos_hourly.py` — 16지점 × 192개월 시간자료 수집
2. `src/impute_asos_hourly.py` — 시간 보간(연속 1-2시간) + 인접 지점 bias 보정
3. `src/build_weather_hourly_nationwide.py` — 시간별 인구가중 전국평균 (TA/HM/WS)
4. `src/build_feels_like.py` — 시간별 PT 산출 → 일 max/min 집계 (단일 산출 파일)
5. (별도) `src/build_extreme_flags.py` — PT 기반 폭염 더미 + ta_min 기반 한파 더미

### 검증 사례 (계산 결과 예측치)

| 사례 | 입력 | 공식 | 기대 PT |
|---|---|---|---|
| 폭염일 (서울 2018-08-01) | T=39°C, RH=60% | 여름 | ~45°C |
| 폭염일 (대구 2024-07-29) | T=37°C, RH=70% | 여름 | ~44°C |
| 한파일 (속초 2023-01-25) | T=-12°C, V=7m/s | 겨울 | ~-22°C |
| 평이한 가을 (10-15) | T=20°C, RH=50%, V=2m/s | 겨울 (조건 미충족) | 20°C (보정 없음) |

→ 산출 후 위 값과 ±0.5°C 이내 일치 여부 확인.

### 한파 임계 더미는 PT 미사용

기상청 한파 특보 ②조건은 "아침 최저 **기온** ≤ -12/-15°C" — **실제 기온
(`ta_min`) 기준**. PT가 아님. 따라서 한파 더미는 PT 도입과 무관하게 `ta_min`로
산출. (`feels_like_min`은 보조 분석용 컬럼으로만 유지)

### 다중공선성 관리

`feels_like_max` ↔ `ta_max` 상관 0.95+ 예상.
→ 두 변수 동시 투입 시 ARIMAX 계수 불안정.
→ **본 모델에서는 `feels_like_max` (체감) 단독 사용**, `ta_max`는 §2.2 기본
변수로 별도 분석 시에만 활용.
