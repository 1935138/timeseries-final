# Research Log

## 2026-05-25 — 달력·공휴일 변수 수집 및 생성

### 대상
**2010-01-01 ~ 2025-12-31 일별 달력 변수(요일·월·주말·공휴일)를 생성한다.**
공휴일은 공공데이터포털 특일 API(한국천문연구원)를 단일 출처로 사용.

### 산출물
- `data/raw/holidays_2010_2025.csv` — API 원시 수집 (275행)
  - 컬럼: `date, name, is_holiday, date_kind`
- `data/processed/calendar_2010_2025.csv` — 일별 달력 wide (5,844행)
  - 컬럼: `tm, dow, month, is_weekend, is_holiday`

### 변수 정의

| 변수 | 정의 | 값 |
|---|---|---|
| `dow` | 요일 | 0=월, 1=화, …, 6=일 (pandas `dayofweek` 기준) |
| `month` | 월 | 1~12 |
| `is_weekend` | 토·일 여부 | 0/1 |
| `is_holiday` | 법정공휴일 여부 (대체·임시공휴일 포함) | 0/1 |

설계 결정:
- 명절(설/추석), 연휴 길이, 샌드위치 데이 등 파생 변수는 생성하지 않음.
  명절도 일반 공휴일과 동일하게 `is_holiday=1`로 통합.
- 대체공휴일·임시공휴일은 특일 API의 `isHoliday=Y` 출력에 그대로 포함됨.

### API 호출 사양

#### 출처
- 공공데이터포털 "한국천문연구원_특일 정보" (data.go.kr/data/15012690/openapi.do)
- 가이드: `docs/OpenAPI활용가이드_한국천문연구원_천문우주정보__특일_정보제공_서비스_v1.4.docx`
- 인증키 환경변수: `DATA_GO_KR_HOLIDAYS_KEY` (.env)
  - 명명 규칙: `DATA_GO_KR_<서비스명>_KEY` (공공데이터포털은 API별 별도 키 필요)

#### 엔드포인트
```
http://apis.data.go.kr/B090041/openapi/service/SpcdeInfoService/getRestDeInfo
```
- 호출 단위: 연 1회 × 16년 = 16회
- 필수 파라미터: `serviceKey, solYear, numOfRows`
- 옵션: `_type=json` (기본은 XML)

### 호출 디버깅 (3단계 시행착오)

#### 1단계: 키 이중 인코딩 (401)
- 초기 시도: `requests.get(url, params={"serviceKey": key, ...})`
- 결과: HTTP 401, 응답 URL에 `%25%2B` (이중 인코딩) 확인
- 원인: 공공데이터포털 Encoding 키는 이미 URL-encoded(`%2B` 등)인데
  requests가 또 인코딩.
- **해결**: serviceKey는 URL 문자열에 직접 삽입, 나머지만 params로:
  ```
  url = f"{API_URL}?serviceKey={key}"
  resp = requests.get(url, params={"solYear": "2024", ...})
  ```

#### 2단계: HTTPS 거부 (403)
- 키 형식은 정상화되었으나 여전히 HTTP 403 Forbidden.
- 단서: 가이드 명세에 서비스 URL이 **HTTP만 명시** (개발·운영 모두).
- 검증: curl로 동일 URL을 `http://`로 호출 → 200 정상 응답.
- **해결**: `API_URL`을 `http://` 로 변경. (공공데이터포털은 일부 API가
  HTTPS 게이트웨이 미지원으로 403 반환)

#### 3단계: 산발적 403 (rate limit 추정)
- HTTP로 변경 후 2010~2012는 성공, 2013에서 403.
- 호출 간격이 짧을 때 게이트웨이가 일시 차단하는 패턴.
- **해결**:
  - 호출 간 sleep 0.3s → **1.0s**
  - 지수 백오프 재시도 (1, 2, 4, 8, 16초, 최대 5회)
- 결과: 16년 모두 성공
