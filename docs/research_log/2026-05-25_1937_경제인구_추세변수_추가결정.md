# Research Log

## 2026-05-25 — 장기 추세 외생변수 (경제·인구) 추가 결정

### 배경

기존 Phase 1 외생변수(기상·달력)만으로는 **2010-2025 종속변수의 구조적 상승
추세**(인구·산업·데이터센터 등 거시 동인에서 기인)를 외생변수로 명시 포착할
수 없다는 문제 제기.

사용자 직관: **"최대전력 예측에 실제 공급능력을 봐야 한다 — 2010보다 2025의
사용량이 인구·데이터센터 등으로 늘었을 것"**

### 분석: "공급능력"은 추세 동인의 대리변수로 부적절

| 항목 | 결론 |
|---|---:|
| 추세를 외생변수로 명시해야 한다는 문제 의식 | **타당** |
| "공급능력"을 그 변수로 쓰는 것 | **부적절** |

**이유 — 인과 방향 역전(reverse causality)**:
```
[진짜 원인]            [관측되는 결과]
GDP/인구/데이터센터  →  수요(peak_mw) 증가
                              ↓
                       발전사·정부 수요 증가 인지
                              ↓
                       발전소 건설 결정 (5~10년 lag)
                              ↓
                       공급능력 증가
```

공급능력은 수요의 **원인이 아니라 결과**. ARIMAX 외생성 가정(`X → y` 단방향)
위배. 또한 공급능력과 `peak_mw`는 둘 다 같은 상승 추세를 공유 → 상관 매우
높지만(r ≈ 0.95+) 회귀계수 해석 불가, forecast 시 순환 의존 문제.

### 결정: Phase 1에 다음 3개 변수 추가

| 변수명 | 정의 | 빈도 | 출처 | 가공 |
|---|---|---|---|---|
| `kosis_ip_total` | 전산업생산지수 (2020=100) | 월 | KOSIS | T+2개월 lag → 일별 ffill |
| `kosis_ip_semicon` | 반도체 출하지수 (or 광공업 내 반도체 부문) | 월 | KOSIS | T+2개월 lag → 일별 ffill, ma3 스무딩 |
| `pop_total` | 전국 주민등록인구 합계 | 연 | 기존 KOSIS 수집 데이터 합산 | 연도값 → 일별 ffill |

**선택 근거**:
- **외생성**: 셋 다 인과 화살표가 `X → y` 방향 (전력 수요가 산업생산·인구를
  결정하지 않음).
- **추세 동인 차별화**:
  - `ip_total`: 경기 사이클 + COVID 충격(2020-03~05) + 회복 시그널
  - `ip_semicon`: 반도체·데이터센터 호황 (한국 산업용 전력의 8~10% 비중)
  - `pop_total`: 완만한 장기 추세 (2020 이후 감소 전환 시점 포착)
- **데이터 가용성**: 모두 KOSIS OpenAPI로 무료, 2010-2025 전 기간 가용.
- **vintage 안전**: 발표 lag(2개월) 강제 적용해 미래 정보 누설 방지.

### 검토했으나 제외

| 변수 | 제외 사유 |
|---|---|
| 일별 공급능력 (`supply_capacity_mw`) | 역인과·다중공선성 |
| 일별 예비율 (`reserve_margin`) | 정의상 `peak_mw`의 함수 — 정보 누설 |
| 에어컨 보급률 | 이미 포화(~97%), 추세 변동 없음 |
| 데이터센터 IT 부하 직접값 | 공식 시계열 미공개 (KDCC 연차보고서 수기 정리 필요) |
| 가구수 | 보조 후보로 보류, 1차에서는 인구만 사용 |
