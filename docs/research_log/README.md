# Research Log

이 디렉토리는 본 프로젝트의 **연구·구현 기록**을 시간 순으로 보관합니다.
각 파일은 하나의 의사결정·실험·구현 단위를 다룹니다.

## 파일명 규칙

```
YYYY-MM-DD_HHMM_주제.md
```

- `YYYY-MM-DD_HHMM` — 작업을 **시작한 시각**(로컬 시간, KST). 작성 시점이 아니라
  실제로 그 작업/실험/결정을 한 시각을 적습니다. 동일 일자 내에서 어떤 순서로
  진행되었는지 추적하기 위함입니다.
- `주제` — 한국어 가능. 공백 대신 `_` 사용. 너무 길지 않게 핵심 키워드 위주로.
  - 좋음: `2026-05-25_1616_ASOS_기상자료_수집.md`
  - 피함: `2026-05-25_1616_오늘_한_일.md`

파일명 정렬만으로 시간 순 흐름이 보이도록 하는 것이 목적입니다.

## 작성 규칙

1. **결론이 문서의 메인입니다.** 위쪽에는 현재 채택된 최종 결정·산출물을,
   아래쪽에는 거기에 도달한 변경 이력을 둡니다.
2. **하나의 파일 = 하나의 결정/작업.** 여러 주제를 한 파일에 섞지 않습니다.
   하루에 여러 건을 했다면 파일을 나눕니다.
3. **같은 파일 내 수정 vs 새 로그 작성.**
   - 같은 작업의 **세부 보정**(파라미터 조정, 1차→2차 판단)은 같은 파일의
     아래쪽 "변경 이력"에 누적합니다. 상단의 최종 결정도 같이 갱신.
   - 결정 자체를 **뒤집는 큰 방향 전환**은 새 로그 파일을 만들고, 원 파일
     머리말에 "→ 후속: `2026-MM-DD_HHMM_*.md` 참조" 한 줄을 추가합니다.
4. **재현 가능성에 필요한 것은 본문에 남깁니다.**
   - 사용한 원자료 파일 경로 / 출처 URL
   - 핵심 파라미터·임계치
   - 산출물 경로 (예: `data/전력수급_2010-2025.csv`)
5. 결과만이 아니라 **왜 그렇게 결정했는지** 를 남깁니다.

## 권장 템플릿

```markdown
# Research Log

## YYYY-MM-DD — <주제 한 줄 요약>

> 이 작업이 최종적으로 무엇을 결정/생성했는지 1–2문장.

### 대상
무엇에 대한 작업인지 (종속변수/독립변수/전처리/모델 등), 데이터·코드 범위.

### 최종 결정
지금 유효한 결정·방법·파라미터. 독자가 이 섹션만 읽어도 현재 상태를
이해할 수 있어야 합니다.

### 산출물 / 적용 범위
- 산출 파일 경로 (예: `data/전력수급_2010-2025.csv`)
- 영향받는 후속 작업·코드

### 후속 작업
다음에 해야 할 일, 미해결 이슈, 의존 작업.

---

### 변경 이력

시간 순(오래된 것이 위).

#### YYYY-MM-DD HH:MM — 1차 판단
처음에는 무엇을 어떻게 하려 했는지, 어떤 근거였는지.

#### YYYY-MM-DD HH:MM — 수정: <무엇을 바꿨나>
어떤 사실·관찰·재검토로 인해 결정을 바꿨는지. 위 "최종 결정" 섹션도
이때 함께 갱신.
```
