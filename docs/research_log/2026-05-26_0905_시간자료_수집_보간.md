# Research Log

## 2026-05-26 — ASOS 시간자료 수집·결측 보간 (체감온도 산출 입력)

### 목적

기상청 공식 체감온도(PT) 산출을 위해 1시간 해상도 ASOS 자료를 수집한다.
일평균 입력 근사 대신 시간별 산출 → 일 max/min 집계가 공식 정의에 정합
(결정 근거: `2026-05-25_2139_체감온도_공식_확정.md`).

### 산출물

- `data/raw/asos_hourly/asos_hourly_stn<지점>_2010_2025.csv` (22개)
  - 컬럼: `tm` (YYYY-MM-DD HH:00), `stn`, `ta`, `hm`, `ws`
  - 평균 140,235행/지점 (기대 140,256, 차이 0.015%)
- `data/processed/asos_hourly_imputed/asos_hourly_stn<지점>_2010_2025_imputed.csv` (16개)
  - 컬럼: `tm`, `stn`, `ta`, `hm`, `ws` + `<col>_src` 출처
  - 잔여 결측 0

| 항목 | 값 |

| 엔드포인트 | `https://apihub.kma.go.kr/api/typ01/url/kma_sfctm3.php` (시간자료 구간조회) |

핵심 컬럼 위치: TM(1), STN(2), WS(4), TA(12), HM(14).

### 보간 처리

방법론은 일자료와 동일 (인접 지점 bias 보정 → 시간 선형 보간).

| 단계 | 처리 |
|---|---|
| 1. 중복 시각 통합 | KMA가 보정·재발표한 케이스 (지점당 ~수십건) 수치 평균 처리 |
| 2. 시간축 완전성 | `pd.date_range(freq='h')` reindex로 누락 시각을 NaN으로 강제 |
| 3. 백업 지점 bias 보정 | bias = mean(main_train) - mean(backup_train), train ≤ 2024-12-31 |
| 4. 시간 선형 보간 | `interpolate(method='time', limit=12)` — 연속 12시간(반나절)까지 |

**bias 단일값**: 시간별 bias는 표본 작아 노이즈 ↑ → 연중 통합 1개 bias 사용 (일자료 방식과 동일).

### 디버깅 이슈 (3건)

#### 1) `kma_sfctm2.php` 단일 시각 응답 한계
- `tm1`·`tm2` 인자가 무시되고 최신 시각 1건만 반환.
- 시각·전국(`stn=0`) 또는 시각·1지점만 1회 응답.
- 16년치를 받으려면 호출 횟수 폭증 (140k 호출/지점).
- **해결**: 활용신청 필요한 `kma_sfctm3.php` (구간조회) 사용.

#### 2) 중복 시각
- 같은 시각에 2건 발행되는 케이스 발생 (서울 9개 시각×2건 등).
- KMA가 보정값을 재발행한 패턴으로 추정. 두 값 차이는 ≤ 0.5°C 수준.
- 그대로 `reindex` 시 `ValueError: cannot reindex on duplicate labels`.
- **해결**: `load_station`에서 `groupby('tm').mean()` 으로 통합.

#### 3) Series → PeriodIndex 변환 오류
- (별도 `build_ip_daily.py`에서 발생, 본 작업과는 무관 — 참고용)
